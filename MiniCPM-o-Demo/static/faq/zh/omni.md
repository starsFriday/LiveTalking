# Omni Full-Duplex 常见问题

## 什么是 Omni Full-Duplex 模式？

Omni 全双工模式下，你可以一边说话一边展示画面，模型能同时看、听并实时回复，不会遗漏任何信息，同时保持低延迟反馈。这是 MiniCPM-o 4.5 最强大的交互模式。

## 需要什么设备？

- **摄像头**：用于实时视频输入（内置或外接均可）
- **麦克风**：用于语音输入
- **耳机（可选）**：避免模型回复的声音被麦克风再次采集，产生回声

## 回复过于简短？

由于模型训练语料有局限性，回复可能较为简洁。**可以通过左侧栏调高 Length Penalty 参数，达到 1.3 实现更长的回复长度和更好的共情能力。**默认值为 1.05 回复较为简洁。但已知问题是在 Length Penalty = 1.3 时，可能难以语音打断。下个模型版本会重点提升。

## 可以自定义音色、韵律吗？

可以，通过Preset System Prompt点击Advanced按钮，上传自定义参考音频即可。MiniCPM-o 4.5 提出了多模态系统提示词概念，可以允许用户在系统提示词中加入 文字系统指令、参考音频等，模型将可以根据文字指令和参考音频，改变回复的文本风格、语音韵律等。

## 可以自定义系统提示词吗？

可以，通过Preset System Prompt点击Advanced按钮，改变系统提示词即可。改变提示词会改变模型风格、行为等。MiniCPM-o 4.5 提出了多模态系统提示词概念，可以允许用户在系统提示词中加入 文字系统指令、参考音频等，模型将可以根据文字指令和参考音频，改变回复的文本风格、语音韵律等。

## 体感卡顿怎么办？

- 如果部署在A100/4090等上一代GPU上，可以开启 `torch.compile` 加速，具体参考[torch.compile 加速](https://openbmb.github.io/minicpm-o-4_5-pytorch-simple-demo/site/zh/compile.html)，可显著提升体感流畅度。每个双工单元(1s)的推理时间可以降低1倍达到0.5s，实现0卡顿。
- 确认服务端 GPU 负载，可在 `/admin` 页面查看 Worker 状态

## Live 模式和 File 模式有什么区别？

- **Live**：使用实时摄像头 + 麦克风输入
- **File**：上传预录制的视频文件作为输入，支持选择音频来源（视频音轨、麦克风、或同时用麦克风和视频音轨混合）

## Vision HD 是什么？

Vision HD 开启后，每帧图像会使用高清视觉编码（192 tok/frame vs 默认 64 tok/frame），适合需要识别文字、细节的场景。但会增加约 100ms 延迟。

## MaxKV 参数是什么意思？

MaxKV 限制 KV 缓存大小（即上下文窗口长度）。默认 8192 tokens，为模型训练长度，超过长度会出现问题。不建议更改。

## 为什么画面显示不出来？

- 确认浏览器已授予 **摄像头权限**
- 部分浏览器（尤其 Firefox）对 HTTPS 有要求，确认使用 HTTPS 访问
- 点击画面右上角的翻转按钮尝试切换前后摄像头
- 检查是否有其他应用正在占用摄像头

## 全屏模式怎么用？

点击视频右下角的 **全屏按钮** 进入全屏模式。全屏下可以通过底部字幕查看对话内容，字幕的高度和透明度可在 **Fullscreen Subtitle** 配置中调整。

## 左侧栏的 Delay 参数有什么用？

播放延迟缓冲，数值越高越**稳定**但延迟越大。默认 200ms。如果遇到音频断续，可以适当调高。网络良好的情况下可以降到 100ms。
